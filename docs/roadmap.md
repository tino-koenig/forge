

# Roadmap

## Current phase — Consolidation

Focus:
- consolidate the existing foundation
- clean up code where needed
- align and improve documentation
- reduce inconsistencies in structure and behavior

Goals:
- make the current implementation more coherent and maintainable
- establish clearer patterns for future work
- prepare the repository for deliberate v1 feature planning

Out of scope:
- unrelated large new features
- broad rewrites without a clear reason
- feature expansion that weakens consistency during consolidation

## Phase 0 — Foundation and framework preparation (completed)

Scope:
- establish the basic project skeleton
- build the first features and working flows
- prepare the framework and technical base for further development

Outcome:
- Forge has an initial working foundation
- core ideas and early behavior are present in the repository
- some parts were optimized for speed of progress rather than consistency

## Phase 1 — Consolidation, cleanup, and documentation (current)

Scope:
- consolidate the existing codebase
- clean up and simplify implementation where appropriate
- improve documentation and align it with the real system
- reduce avoidable inconsistency in patterns, naming, and structure

Outcome target:
- a more stable and understandable repository
- clearer conventions for contributors and coding agents
- a better base for defining Forge v1 deliberately instead of growing it ad hoc

## Phase 2 — Forge v1 planning (next)

Scope:
- define what belongs into Forge v1
- decide which features are required, optional, or out of scope
- turn broad ideas into concrete, reviewable feature definitions
- prioritize the work for the next implementation stage

Expected output:
- a clearly defined Forge v1 scope
- prioritized feature plans
- a better implementation sequence for the next stage

## Phase 3 — Forge v1 implementation (planned)

Scope:
- implement the selected v1 features
- keep implementation aligned with the clarified project direction
- improve stability, consistency, and release readiness while building out v1

Outcome target:
- a coherent Forge v1 feature set
- implementation that reflects deliberate planning rather than opportunistic growth
- a repository that is closer to a clear v1 release state
