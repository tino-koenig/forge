# User Docs Contribution Workflow (DE/EN)

## Workflow

1. Update the primary page in one locale.
2. Update the counterpart in the other locale.
3. Verify internal links and section parity.
4. Submit PR with the checklist below.

## PR Checklist

- EN page updated
- DE page updated (or explicitly marked pending)
- Links checked in both locales
- Command examples verified against current CLI help
- Trust/Safety wording still matches current capability boundaries

## Required Locale Pairs

Each page under `docs/user/en/` must have an equivalent in `docs/user/de/`.

## Notes

- Keep user docs concise and operational.
- Move deep implementation details to feature/issue docs and link to them.
